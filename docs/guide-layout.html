1. FOUNDATION - Container Classes

Container Building Blocks

These are the foundational CSS classes that every documentation section uses. Master these first.

Required Container Classes

.section-wrapper REQUIRED: Outer container with border and background for every section
.section-label REQUIRED: Top identifier bar (like "1. FOUNDATION - Container Classes")
.component-body SIMPLE CONTENT: Main content area with padding
.component-section COMPLEX CONTENT: Full section with separate header + body
.component-header SECTION HEADERS: Title and description area
.example-wrapper EXAMPLES: Individual example containers within grids

3. FOUNDATION - Grid Layout Classes

Layout Grid Building Blocks

These grid classes organize content within your containers. Learn these to create structured layouts.

.token-grid TOKEN GRID: Specialized grid for design token references

12. ADVANCED - Template Usage Rules

Template Enforcement Rules

Every design system documentation page must use these template patterns for consistency.

Required Template Structure

.section-wrapper REQUIRED - Outer container for every documentation section
.section-label REQUIRED - Must identify section type (Components, Tokens, etc.)
.component-section OR .component-body REQUIRED - Use component-section for major components, component-body for simple content
.example-wrapper REQUIRED - For all interactive examples and code samples
.variants-grid REQUIRED - When showing multiple component states or examples

Template Selection Guide

Major Component Documentation

Use: .component-section with .component-header + .component-body

For: Buttons, Forms, Navigation, Complex Components

Simple Content Sections

Use: .component-body directly

For: Token references, Guidelines, Simple lists

Interactive Demonstrations

Use: Interactive Component Pattern

For: Live demos, Playground areas, State showcases

Code Examples

Use: Code Examples Pattern

For: HTML/CSS snippets, Implementation guides
